# Phase 06.1: Vercel Gmail Configuration - Research

**Researched:** 2026-03-18
**Domain:** Vercel Environment Variables / Infrastructure Configuration
**Confidence:** HIGH

## Summary

This is an infrastructure/configuration phase requiring no code changes. The goal is to sync Gmail credentials (GMAIL_USER and GMAIL_APP_PASSWORD) from local .env.local to Vercel environment variables. The enquiry form API route at `/api/enquiry` uses nodemailer with Gmail SMTP and requires these environment variables to send emails. Currently, the local development environment works, but the production Vercel deployment is missing these credentials, causing email functionality to fail.

**Primary recommendation:** Use Vercel CLI to add environment variables to Production environment: `vercel env add GMAIL_USER production` and `vercel env add GMAIL_APP_PASSWORD production`, then trigger a redeploy.

## Common Pitfalls

### Pitfall 1: Forgetting to Redeploy
**What goes wrong:** Environment variables are only applied to new deployments. Adding them does not automatically trigger a redeploy.
**How to avoid:** After adding environment variables, manually trigger a redeploy via:
- Push a commit to trigger auto-deploy, or
- Use Vercel Dashboard to redeploy
**Verification:** Check Vercel deployment status after adding env vars

### Pitfall 2: Using Regular Password Instead of App Password
**What goes wrong:** Gmail requires an App Password, not the regular account password. Using the wrong password will cause authentication failures.
**How to avoid:** The local .env.local already has the correct App Password (`wyxa ktwu odib hqbu`). Use this exact value when syncing to Vercel.
**Warning signs:** Error message "535 5.7.8 Credentials not accepted"

### Pitfall 3: Adding to Wrong Environment
**What goes wrong:** Adding variables to Development instead of Production means production deployment won't have access to them.
**How to avoid:** Specify `production` explicitly in CLI command or select Production in dashboard dropdown

### Pitfall 4: Not Marking Password as Sensitive
**What goes wrong:** While not strictly required, the app password should be marked sensitive to hide it in the dashboard.
**How to avoid:** Use `--sensitive` flag with CLI or check "Sensitive" checkbox in dashboard

## Code Examples

### Enquiry API Route (What Uses These Variables)
```typescript
// Source: app/api/enquiry/route.ts (lines 5-17)
async function getTransporter() {
  if (!process.env.GMAIL_USER || !process.env.GMAIL_APP_PASSWORD) {
    throw new Error('Gmail credentials not configured. Please set GMAIL_USER and GMAIL_APP_PASSWORD in .env.local')
  }
  const nodemailer = await import('nodemailer')
  return nodemailer.default.createTransport({
    service: 'gmail',
    auth: {
      user: process.env.GMAIL_USER,
      pass: process.env.GMAIL_APP_PASSWORD,
    },
  })
}
```
