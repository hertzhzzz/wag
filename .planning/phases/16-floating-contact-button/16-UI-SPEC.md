## Component Specifications

### Floating Contact Button

| Property | Desktop | Mobile |
|----------|---------|--------|
| Size | 56px × 56px | 40px × 40px |
| Position | fixed, bottom-right, 20px from edges | same |
| Background | Navy (#0F2D5E) | Navy (#0F2D5E) |
| Border-radius | full (circle) | full (circle) |
| Icon | Mail (lucide-react), 24px | Mail, 18px |
| Text | "Contact Us", 14px, white, below icon | hidden |
| z-index | 9999 | 9999 |

**Animation — Pulse Ring:**
- Outer ring: 64px × 64px (desktop), 48px × 48px (mobile)
- Ring color: Navy (#0F2D5E) at 50% opacity
- Ring animation: scale 1.0 → 1.15 → 1.0, opacity 1.0 → 0.5 → 1.0
- Duration: 2s, infinite, ease-in-out
- The button body is inside the ring (ring is a separate element behind)

**Animation — Hover:**
- Button: `scale(1.05)`, shadow deepens (`shadow-lg` → `shadow-xl`)
- Duration: 150ms ease-out
- Transition: `transform 150ms ease-out, box-shadow 150ms ease-out`

**Animation — Reduced Motion:**
- If `prefers-reduced-motion: reduce`, pulse ring animation is disabled
- Hover animation remains (transform is safe)

**ARIA Label:** `"Open contact form"` on the button element

### Contact Modal Overlay

| Property | Value |
|----------|-------|
| Background | Navy/60 (`rgba(15, 45, 94, 0.6)`) + `backdrop-blur-sm` |
| z-index | 9999 (same as button) |
| Close triggers | ESC key, overlay click, X button |

**Animation — Modal Enter:**
- Type: fade-in + zoom-in-95
- Duration: 200ms
- Easing: ease-out
- Starting scale: 0.95 → 1.0
- Starting opacity: 0 → 1

**Animation — Modal Exit:**
- Duration: 150ms
- Type: fade-out + zoom-out-95

### Modal Dialog

| Property | Value |
|----------|-------|
| Background | white |
| Border-radius | 1rem (16px) |
| Max-width | 28rem (448px) |
| Padding | 2rem (32px) desktop, 1.5rem (24px) mobile |
| Shadow | `shadow-2xl` |
| Position | relative, centered via flex parent |

**Modal Header:**
- Heading: "Contact Us", 20px, weight 600, Navy
- Close button: top-right, 40px × 40px, X icon, hover `bg-navy/5`

### Contact Form

**Fields:**

| Field | Type | Placeholder | Required | Validation |
|-------|------|-------------|----------|------------|
| Email | email input | "your@email.com" | Yes | Zod email schema |
| Message | textarea | "How can we help you?" | Yes | min 10 chars, max 2000 |

**Field Styling:**
- Background: white
- Border: `1px solid rgba(15, 45, 94, 0.1)`
- Border radius: 0.75rem (12px)
- Padding: 12px 16px
- Focus: `border-amber`, `ring-2 ring-amber/20`
- Placeholder: Navy/30

**Textarea:** 4 rows minimum, auto-expand optional, resize vertical only

**Submit Button:**
- Label: "Send Message"
- Background: Amber (#F59E0B)
- Text: white, 16px, weight 600
- Width: full-width
- Border-radius: 0.75rem (12px)
- Padding: 12px 24px
- Hover: `bg-amber/90`
- Disabled: opacity 50%, cursor not-allowed
- Loading state: text changes to "Sending..."

**Error State:**
- Red (#EF4444) text, 14px
- Displayed below the relevant field
- Icon:AlertCircle (lucide-react) optional

**Success State:**
- Modal content replaced with centered check icon (amber background circle)
- Heading: "Message Sent"
- Body: "We will get back to you within 24 hours."
- Button: "Close", Navy background, white text

## Interaction States

### Button States
| State | Visual |
|-------|--------|
| Default | Navy circle, Mail icon, "Contact Us" label, pulse ring animating |
| Hover | scale(1.05), shadow-xl, pulse ring continues |
| Active/Pressed | scale(0.98) |
| Focus-visible | 2px amber outline offset 2px |
| Modal open | Button hidden or button stays visible (planner decision — not specified) |

### Form States
| State | Visual |
|-------|--------|
| Idle | Empty fields with placeholders |
| Typing | Field border normal, no error |
| Blurred (invalid) | Red border, error message below |
| Submitting | Submit button disabled, "Sending..." |
| Submitted | Success state replaces form |
| Server error | Error message, form stays open, fields retain values |

### Modal States
| State | Visual |
|-------|--------|
| Opening | fade-in + zoom-in-95, 200ms |
| Open | Fully visible |
| Closing | fade-out + zoom-out-95, 150ms |
| Closed | Unmounted from DOM |

## Accessibility

| Requirement | Implementation |
|-------------|---------------|
| Focus trap | When modal open, focus moves inside; Tab cycles within modal |
| ESC to close | `keydown` listener on document |
| ARIA label on button | `aria-label="Open contact form"` |
| ARIA label on close | `aria-label="Close modal"` |
| Focus visible | Custom focus ring: `ring-2 ring-amber ring-offset-2` |
| Reduced motion | Pulse animation disabled via `@media (prefers-reduced-motion: reduce)` |
| Screen reader | Modal uses `role="dialog"`, `aria-modal="true"`, `aria-labelledby` pointing to heading |
| Color contrast | All text meets WCAG AA (4.5:1 for body, 3:1 for large text) |

## Technical Integration Points

| Item | Value |
|------|-------|
| Placement | `app/layout.tsx` — as a client component mounted once |
| z-index | 9999 (below Navbar's z-50 or z-100, above page content) |
| API endpoint | POST `/api/enquiry` |
| API payload | Planner must resolve: API expects `fullName`, `companyName`, `industry`, `lookingFor`; form sends only `email` + `message`. Flagged for plan-phase resolution. |

**API payload mismatch — PLANNER ACTION REQUIRED:**
The existing `/api/enquiry` route validates with `fullName`, `companyName`, `email`, `phone`, `industry`, `customIndustry`, `lookingFor` (all required except phone). The floating contact form per CONTEXT.md only collects `email` + `message`. Planner must decide: (a) create a new lightweight `/api/contact` endpoint, (b) modify enquiry schema to be optional for floating button, or (c) collect minimal Name field.
