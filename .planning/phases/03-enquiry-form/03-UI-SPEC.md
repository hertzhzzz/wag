## Form-Specific Mobile Requirements

### RESP-04: Responsive Layout

| Breakpoint | Layout |
|------------|--------|
| < lg | Single column, stacked |
| >= lg | Two columns (Calendly left, form right) |

**Form container padding:** px-4 sm:px-8 (consistent with Phase 2)

### FORM-01: Keyboard Avoidance

Strategy: Use viewport-relative positioning and scroll-into-view
- Add `scrollIntoView({ behavior: 'smooth', block: 'center' })` on input focus
- Ensure form container has `scroll-margin-top` for anchor offsets
- Add bottom padding (mb-[20vh] or similar) to allow scrolling past submit button

### FORM-02: Label Visibility on Focus

- Labels already use `mb-1.5` (6px) margin — verify sufficient on mobile
- Add `sticky` labels for long forms: `position: sticky; top: 0; bg-white`
- On focus: ensure label remains visible above keyboard

### FORM-03: Submit Button Accessibility

- Button already uses `w-full py-3.5` (>= 44px)
- Add bottom padding to form container to ensure button visible above keyboard
- Consider sticky "Submit" button on mobile after form completion
