<objective>
Deploy website to Vercel production with custom domain and configure environment variables.

Purpose: Make website publicly accessible at winningadventure.com.au with HTTPS.

Output: Live website at winningadventure.com.au with working SSL and environment variables.
</objective>

<context>
@frontend/vercel.json

## Environment Variables Required

From research and .env.local, the following variables are needed:
- NEXT_PUBLIC_SUPABASE_URL
- NEXT_PUBLIC_SUPABASE_ANON_KEY
- RESEND_API_KEY

## Vercel Deployment Steps

1. Import GitHub repository to Vercel
2. Configure environment variables in Vercel dashboard
3. Deploy to production
4. Add custom domain winningadventure.com.au
5. Verify SSL certificate is issued
</context>

<task type="checkpoint:human-action">
  <name>Task 3: Configure environment variables in Vercel</name>
  <action>
    In Vercel dashboard for your project:

    1. Go to Settings > Environment Variables
    2. Add each variable from .env.local:
       - NEXT_PUBLIC_SUPABASE_URL = [value from Task 2]
       - NEXT_PUBLIC_SUPABASE_ANON_KEY = [value from Task 2]
       - RESEND_API_KEY = [value from Task 2]
    3. Set scope: Production (and Preview, Development)
    4. Click Save

    After adding env vars, trigger a deployment:
    - Go to Deployments
    - Click "Deploy" or push a commit to trigger auto-deploy
  </action>
  <how-to-verify>
    Environment variables added in Vercel dashboard, deployment triggered
  </how-to-verify>

</task>

<task type="checkpoint:human-action">
  <name>Task 4: Configure custom domain and SSL</name>
  <action>
    In Vercel dashboard:

    1. Go to Settings > Domains
    2. Enter: winningadventure.com.au
    3. Click Add

    Vercel will provide DNS records:
    - ALIAS or A record pointing to Vercel
    - For .au domain, you may need to configure at your domain registrar

    4. After adding domain, Vercel auto-provisions SSL certificate
    5. Wait for propagation (may take up to 24-48 hours)

    Verify:
    - Visit https://winningadventure.com.au
    - Check SSL certificate is valid
  </action>
  <how-to-verify>
