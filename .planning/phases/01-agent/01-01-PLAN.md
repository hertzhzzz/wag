```markdown
# WAG AI Native - Checkpoint Protocol

## State Storage Location
- Root: `~/.claude/teams/wag-ai/state/`
- Checkpoints: `~/.claude/teams/wag-ai/checkpoints/`

## State Schema (orchestrator.json)

| Field | Type | Description |
|-------|------|-------------|
| currentTask | string | Current task being executed |
| progress.completed | number | Number of tasks completed |
| progress.total | number | Total tasks in phase (6 for Phase 1) |
| progress.successful | number | Number of successful tasks |
| errors | array | Error messages from failed tasks |
| nextSteps | array | Pending actions |
| checkpoints | array | Checkpoint history |
| taskHistory | array | Task execution records |

## Checkpoint Frequency (per D-04)
1. **Hourly auto-save**: Every 60 minutes during active work
2. **After key tasks**: Immediately after each validation task completes

## Checkpoint File Naming
Format: `YYYY-MM-DD-HHMM.json` (e.g., `2026-04-09-1430.json`)

## Writing a Checkpoint

```bash
# Read current state
STATE=$(cat ~/.claude/teams/wag-ai/state/orchestrator.json)

# Update state (using jq or manual edit)
# ... modify state ...

# Write checkpoint with timestamp
CHECKPOINT_FILE="~/.claude/teams/wag-ai/checkpoints/$(date +%Y-%m-%d-%H%M).json"
cp ~/.claude/teams/wag-ai/state/orchestrator.json "$CHECKPOINT_FILE"

# Update orchestrator.json with new state
jq '.checkpoints += ["'"$(basename $CHECKPOINT_FILE .json)'"' | {file: ., timestamp: now}]' ~/.claude/teams/wag-ai/state/orchestrator.json > /tmp/orchestrator.json && mv /tmp/orchestrator.json ~/.claude/teams/wag-ai/state/orchestrator.json
```

## Task Completion Recording

After each validation task, update taskHistory:
```json
{
  "taskId": "DevOps-1",
  "taskName": "Modify Navbar CTA button style",
  "intent": "Change Navbar CTA from 'Get Quote' to styled button",
  "actions": ["Modified app/components/Navbar.tsx"],
  "output": "Navbar CTA styled with amber background",
  "score": 1,
  "completedAt": "2026-04-09T14:30:00Z"
}
```

## Accuracy Calculation

Accuracy = (successful / total) * 100%

For Phase 1: 45% threshold = 3 of 6 tasks must succeed

| Score | Meaning |
|-------|---------|
| 1 | Task completed fully as intended |
| 0.5 | Task completed partially |
| 0 | Task failed completely |
