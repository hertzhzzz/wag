## Deep Research Findings

### ⚠️ Critical Issues Found

#### 1. Hero Component - Video Over Image
**File:** `app/components/Hero.tsx`

**Current State:**
- Image has `priority={true}` ✅
- But there's a **video element covering the image** (lines 20-36)
- Video has `preload="metadata"` - this may be the actual LCP element!

**Bug Risk:** HIGH
- The image priority won't help if video is blocking it
- Video poster loads but video itself delays LCP on mobile

**Fix Required:**
- Option A: Remove video on mobile, show image only
- Option B: Add `priority` to video poster, defer video loading

#### 2. Calendly - Not Using Dynamic Import
**File:** `app/enquiry/EnquiryForm.tsx`

**Current State:**
- Uses `useEffect` to load Calendly script (lines 14-21)
- Loads on every page render

**Bug Risk:** MEDIUM
- Should use `next/dynamic` for better code splitting
- Current approach works but is suboptimal

#### 3. Coverage Component - Already Optimized ✅
**File:** `app/components/Coverage.tsx`

**Current State:**
- Uses `dynamic(() => import('react-globe.gl'), { ssr: false })` ✅

**No changes needed!**

## Verified Safe Changes

### ✅ Sharp Installation
- **Status:** Safe - optional peer dependency
- **Installation:** `npm install sharp`
- **Next.js auto-detects:** No config changes needed
- **Build time:** May increase slightly, but production is faster

### ✅ fetchpriority Attribute
- **Status:** Safe - Next.js auto-adds when `priority={true}`
- **Browser support:** Baseline 2024 (Oct 2024) - works on modern browsers
- **Fallback:** Browsers without support ignore it gracefully

### ✅ Bundle Analyzer
- **Status:** Safe - dev-only dependency
- **Installation:** `npm install --save-dev @next/bundle-analyzer`
- **No production impact**
