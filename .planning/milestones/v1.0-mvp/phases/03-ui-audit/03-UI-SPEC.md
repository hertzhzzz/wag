## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing |
| md | 16px | Default element spacing, mobile side padding |
| lg | 24px | Section padding |
| xl | 32px | Layout gaps, desktop side padding |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions:
- Touch targets: 44px minimum height (already in globals.css as `.touch-target`)
- Form inputs: py-3 (approximately 44px) - from enquiry/page.tsx
- Mobile hamburger menu: full-screen overlay

Source: Established in Phase 1-2, globals.css

## Mobile Form Patterns

For Phase 3 focus on enquiry form mobile adaptation:

| Pattern | Implementation |
|---------|----------------|
| Input focus scroll | scrollIntoView({ behavior: 'smooth', block: 'center' }) on focus |
| Label visibility | Labels always visible above input, not placeholder-only |
| Keyboard avoidance | Use padding-bottom on form container or Viewport API |
| Touch targets | All inputs py-3 (44px min), buttons min-h-11 |
| Sticky submit | Submit button always accessible - no scroll required |

Source: REQUIREMENTS.md FORM-01, FORM-02, FORM-03

## Phase 3 Specifics

### Scope

This phase focuses on comprehensive UI audit across all pages:

| Page | Focus Areas |
|------|-------------|
| Home | Verify existing responsive patterns |
| Services | Verify existing responsive patterns |
| About | Verify existing responsive patterns |
| Enquiry | Mobile form adaptation (RESP-04, FORM-01, FORM-02, FORM-03) |
| Resources | Check for mobile issues |

### Success Criteria

1. All pages (Home, Services, About, Enquiry, Resources) have consistent mobile layout
2. No horizontal scroll at 320px width on any page
3. All touch targets meet 44px minimum
4. Navigation works correctly on all mobile pages
5. Forms are usable on mobile (keyboard doesn't cover inputs)

Source: ROADMAP.md Phase 3
