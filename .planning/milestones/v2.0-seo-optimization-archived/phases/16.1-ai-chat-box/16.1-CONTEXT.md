# Phase 16.1: AI Chat Box - Context

**Gathered:** 2026-03-21
**Status:** Ready for planning

<domain>
## Phase Boundary

优化现有 AIChatBox.tsx 的用户体验。这是一个预设选项引导的"假 AI"聊天组件，通过选项流程引导用户填写联系表单，提交到 /api/contact。

**关键决策：删除 FloatingContactButton** — AI Chat Box 成为唯一的悬浮联系入口，Phase 16 的 FloatingContactButton 不再需要。

</domain>

<decisions>
## Implementation Decisions

### Launcher 按钮设计
- **脉冲动画**: 需要，与 FloatingContactButton（已删除）区分
- **图标**: Bot 或 Sparkles（AI 相关图标，突出"AI 助手"概念）
- **颜色**: Navy (#0F2D5E)
- **尺寸**: 56px (w-14 h-14)

### 用户流程优化
- **表单触发**: 用户主动触发，不自动跳转
  - selected 状态后，显示"获取详情"按钮
  - 用户点击按钮后才进入表单
- **提交后操作**: 显示"查看更多"链接，引导到 /resources 或 /services
- **欢迎语**: 更具体，列出具体服务名称
  - 如: "Hi! I can help with Factory Discovery, Pricing, Partnerships..."
- **服务选项**: 调整文案，使更吸引人或清晰

### 移动端体验
- **宽度**: 保持 w-80
- **位置**: bottom-right
- **边距**: 增大到 24-28px，防止误触
- **键盘交互**: 键盘弹出时自动滚动到可见的表单字段

### 动画与状态反馈
- **打字指示器**: 保持 bounce 样式（三个圆点 + delay）
- **展开动画**: 保持当前 opacity + translate transition
- **Loading 状态**: 进度指示，显示更详细的状态文字
- **错误提示**: Toast 样式，顶部弹出，自动消失

### 重大决策：删除 FloatingContactButton
- AI Chat Box 是唯一的悬浮联系入口
- FloatingContactButton (Phase 16) 需要被移除
- 需要从 layout.tsx 中删除 FloatingContactButton 组件

</decisions>
