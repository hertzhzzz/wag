# Phase 16.1: AI Chat Box UX Optimization - Research

**Researched:** 2026-03-21
**Domain:** React/Next.js UI Component Optimization
**Confidence:** HIGH

## Summary

Phase 16.1 focuses on optimizing the existing AIChatBox.tsx component, a "fake AI" chat widget that guides users through a preset option flow to collect contact information. The key change is making the form transition user-initiated (instead of auto-advancing after 2 seconds) and improving the visual design with a pulse animation and better icon.

**Primary recommendation:** Implement user-initiated form trigger with explicit "Get Details" button, add pulse animation to launcher, and use Bot/Sparkles icon for AI differentiation.

### Pattern 1: State Machine Chat Flow
**What:** Chat component uses discrete states (idle → greeting → options → selected → form → submitted)
**When to use:** Guided conversation UX with preset options
**Current implementation:** Already exists in AIChatBox.tsx lines 6, 43-50
**Modification needed:** Add intermediate state or button to delay form transition

```typescript
// Current flow (auto-transition):
handleSelectOption → setSelectedOption → 2s timeout → setChatState('form')

// New flow (user-initiated):
handleSelectOption → setSelectedOption → show "Get Details" button
User clicks "Get Details" → setChatState('form')
```

### Pattern 2: Click-Outside-to-Close
**What:** Chat minimizes when clicking outside the component
**When to use:** Floating widgets that should be dismissible
**Current implementation:** Already exists in AIChatBox.tsx lines 53-65

### Pattern 3: Toast Notification
**What:** Error messages appear at top of viewport, auto-dismiss after delay
**When to use:** Non-blocking error feedback
**Implementation:** Inline toast with setTimeout auto-dismiss (no external library needed)

### Anti-Patterns to Avoid
- **Auto-advancing UX:** Don't automatically transition from selected to form state — users feel rushed
- **Removing click-outside:** The current dismiss-on-click-outside pattern should be kept
- **Using MessageCircle icon:** CONTEXT specifies Bot or Sparkles for AI differentiation

## Common Pitfalls

### Pitfall 1: Mobile Keyboard Scroll
**What goes wrong:** Form fields hidden behind keyboard on mobile devices
**Why it happens:** Viewport doesn't scroll automatically when keyboard opens
**How to avoid:** Use `scrollIntoView({ behavior: 'smooth', block: 'center' })` on form container when chatState becomes 'form'
**Warning signs:** Form renders but input fields not visible on mobile

### Pitfall 2: Pulse Animation Performance
**What goes wrong:** Pulse ring causes janky animations or high CPU usage
**Why it happens:** Incorrect animation implementation (transform/opacity vs layout properties)
**How to avoid:** Use CSS `animation` with `opacity` or `transform` only — never width/height/margin
**Reference:** FloatingContactButton.tsx line 72 uses `-m-4` with absolute positioning, which works

### Pitfall 3: Auto-Transition Feels Rushed
**What goes wrong:** Users complain the chat advances too fast without their input
**Why it happens:** 2-second timeout automatically transitions from selected to form
**How to avoid:** Remove timeout, add explicit "Get Details" button (per CONTEXT decision)

### Pitfall 4: Form Double-Submission
**What goes wrong:** Users click submit twice, causing duplicate messages
**Why it happens:** No loading state prevention during API call
**Current mitigation:** Already has `disabled={loading}` on submit button (line 246)

## Code Examples

### Pulse Animation Implementation
From FloatingContactButton.tsx lines 70-73:
```tsx
{/* Pulse Ring */}
<div className="absolute inset-0 -m-4 hidden sm:block">
  <div className="w-16 h-16 mx-auto my-4 border-2 border-navy/50 rounded-full animate-pulse-ring" />
</div>
```

**Note:** `animate-pulse-ring` is a custom keyframe. Verify it exists in globals.css or add it:
```css
@keyframes pulse-ring {
  0%, 100% { opacity: 0.5; transform: scale(1); }
  50% { opacity: 1; transform: scale(1.05); }
}
.animate-pulse-ring {
  animation: pulse-ring 2s ease-in-out infinite;
}
```

### Keyboard Scroll on Form Open
```tsx
const formRef = useRef<HTMLFormElement>(null)

useEffect(() => {
  if (chatState === 'form' && formRef.current) {
    formRef.current.scrollIntoView({ behavior: 'smooth', block: 'center' })
  }
}, [chatState])
```

### Toast Auto-Dismiss Pattern
```tsx
const [toastError, setToastError] = useState('')

useEffect(() => {
  if (toastError) {
    const timer = setTimeout(() => setToastError(''), 4000)
    return () => clearTimeout(timer)
  }
}, [toastError])

// Render toast at top of chat box:
{toastError && (
  <div className="absolute top-0 left-0 right-0 bg-red-500 text-white text-xs p-2 rounded-t-xl">
    {toastError}
  </div>
)}
```
