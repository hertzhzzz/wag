# Plan 19-03: Page SEO Optimization

## Objective
Optimize all 5 pages for SEO with unique title tags, meta descriptions, proper heading hierarchy, target keywords placement, image alt attributes, and Australian city GEO keywords.

## Tasks

### Task 1: Audit and optimize title tags and meta descriptions (all 5 pages)
<read_first>
- app/page.tsx
- app/services/page.tsx
- app/about/page.tsx
- app/resources/page.tsx
- app/enquiry/page.tsx
</read_first>
<action>
For each page, update the Next.js metadata export to include unique, keyword-optimized title and description.

**Required character counts:**
- Title: 50-60 characters
- Description: 150-160 characters

**Page-by-page metadata:**

**1. Homepage (`app/page.tsx`)**:
```tsx
export const metadata: Metadata = {
  title: 'China Sourcing Company | Australian Business Guide',
  description: 'Connect with verified Chinese manufacturers. Factory tours, supplier verification, and procurement support for Australian businesses since 2012.',
}
```

**2. Services (`app/services/page.tsx`)**:
```tsx
export const metadata: Metadata = {
  title: 'China Factory Visit Services | Supplier Verification',
  description: 'Join guided factory tours in Shenzhen, Guangzhou, and Shanghai. WAG helps Australian businesses verify suppliers and negotiate with manufacturers.',
}
```

**3. About (`app/about/page.tsx`)**:
```tsx
export const metadata: Metadata = {
  title: 'About Winning Adventure Global | Adelaide China Sourcing',
  description: 'Founded in Adelaide, SA. 12+ years helping Australian businesses find reliable China manufacturers. Expert sourcing agents in Shenzhen and beyond.',
}
```

**4. Resources (`app/resources/page.tsx`)**:
```tsx
export const metadata: Metadata = {
  title: 'China Sourcing Resources | Factory Guide for Australians',
  description: 'Free guides on verifying Chinese suppliers, import regulations, and procurement strategies. Expert tips for Australian businesses sourcing from China.',
}
```

**5. Enquiry (`app/enquiry/page.tsx`)**:
```tsx
export const metadata: Metadata = {
  title: 'Contact WAG | Request a Consultation',
  description: 'Get in touch for China sourcing consultation. Factory tours, supplier verification, and bulk procurement support for Australian businesses.',
}
```

Also add `keywords` metadata array to each page with relevant keywords.
</action>
<acceptance_criteria>
- [ ] `grep -E "title:.*50|60" app/page.tsx` (verify title length)
- [ ] `grep "description:.*150|160" app/page.tsx` (verify description length)
- [ ] Each page has unique title (no duplicates across pages)
- [ ] `grep "keywords" app/services/page.tsx` contains Australian cities
</acceptance_criteria>

### Task 2: Fix H1/H2/H3 heading hierarchy on each page
<read_first>
- app/page.tsx
- app/services/page.tsx
- app/about/page.tsx
</read_first>
<action>
Verify and fix heading hierarchy on each page. Each page must have exactly ONE H1, with H2 for major sections, H3 for subsections.

**Homepage (`app/page.tsx`):**
- H1 should be the main hero headline (e.g., "Your Gateway to Verified Chinese Manufacturers")
- H2 for section titles (e.g., "How It Works", "Our Services", "Factory Directory")
- H3 for subsection titles within sections

**Services (`app/services/page.tsx`):**
- H1: "China Factory Visit Services" or similar
- H2 for each service type
- H3 for service details

**About (`app/about/page.tsx`):**
- H1: "About Winning Adventure Global"
- H2 for sections: "Our Story", "Our Mission", "Why Choose Us"
- H3 within sections

**Resources (`app/resources/page.tsx`):**
- H1: "China Sourcing Resources"
- H2 for categories if applicable

**Enquiry (`app/enquiry/page.tsx`):**
- H1: "Get in Touch" or "Request a Consultation"
- H2 for sections: "Contact Form", "Alternative Contact Methods"

**Ensure:**
- No multiple H1s on a single page
- Logical nesting (H2 after H1, H3 after H2)
- Keywords included in headings where natural
</action>
<acceptance_criteria>
- [ ] `grep -c "<h1" app/page.tsx` returns 1
- [ ] `grep -c "<h1" app/services/page.tsx` returns 1
- [ ] `grep -c "<h1" app/about/page.tsx` returns 1
- [ ] `grep "<h2" app/page.tsx | wc -l` returns reasonable count (sections)
- [ ] No `<h3>` appears before an `<h2>` on same page
</acceptance_criteria>
