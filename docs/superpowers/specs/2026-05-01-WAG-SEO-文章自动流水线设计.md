## 1. 背景与目标

**业务背景**：Winning Adventure Global (WAG) 帮助澳大利亚企业从中国采购，需要持续输出高质量 SEO 内容建立信任。

**目标**：建立全自动 SEO 文章流水线，定时触发，持续提升搜索排名。

## 4. 关键词发现

### 数据源（三合一）

| 来源 | 用途 |
|------|------|
| **Google Keyword Planner API** | 搜索量、趋势、相关关键词（需 Google Ads 账号，免费） |
| **Google Trends** | 实时趋势、热点话题、飙升词（免费） |
| **Exa API** | 竞品监控、行业研究、话题发现（MCP 150次/天） |
| **Semantic Scholar API** | 学术研究引用，提升 E-E-A-T（免费，100M+ 论文） |

### 竞品监控范围

- **已知竞品列表**：手动定义 5-10 个竞争对手域名
- **GSC 排名发现**：在你目标词上排名靠前的网站
- **行业通用竞争对手**：搜索结果排名靠前的网站
- **Exa 自动扩展**：发现更多相关网站

### 关键词优先级排序

MASTER Agent 综合以下因素排序：
1. Keyword Planner 搜索量（高优先）
2. Google Trends 趋势（上升中优先）
3. 竞品覆盖差距（你有机会超越的词优先）
4. 长尾匹配度（与 WAG 业务相关度）

## 5. Agent 职责

| Agent | 职责 | 使用工具 |
|-------|------|----------|
| **MASTER** | 关键词优先级排序 → 任务分配 → 流程控制 | Keyword Planner, Trends, Exa, Claude Code |
| **RESEARCHER 1-2** | 并行研究（Exa 搜索 + 学术搜索） | Exa, Semantic Scholar API |
| **DRAFTER** | 合并研究 → 生成 MDX（低AI率优化） | Claude API, 本地写入 |
| **PUBLISHER** | 创建 MDX → commit → push → Google 通知 | Git, Vercel, GSC API, Indexing API |

## 6. 质量门槛

文章必须满足以下所有条件才能发布：

| 检查项 | 标准 | 不通过处理 |
|--------|------|-----------|
| **AI 率检测** | < 30%（GPTZero、Turnitin 等验证） | 自动重写，最多 3 次，仍失败通知你 |
| **内容长度** | 基础 1500+ 字，深度指南 2000+ 字 | 不满足 → 重写 |
| **关键词覆盖** | 主关键词 + 至少 3 个长尾词 | 不满足 → 重写 |
| **原创度检查** | Exa 相似搜索 + 本地 diff-match-patch 比对，重复率 < 20% | 不满足 → 重写 |

---

## 7. 低 AI 率策略

每篇文章必须：

1. **包含真实数据/案例** — 从 Exa 搜索获取最新行业数据
2. **混合句式长度** — 长短句交错（高 burstiness）
3. **融入 WAG 具体服务细节** — 真实的服务描述、价格范围、流程
4. **使用 hedging 语言** — "I think," "in my experience," "honestly"
5. **打破公式化结构** — 不是每个 H2 都按"定义 → 好处 → 步骤 → 总结"
6. **保留引用链接** — 数据来源超链接，提升 E-E-A-T
7. **时效性内容** — Exa 实时搜索确保内容最新（6个月内）

## 9. 错误处理策略

### 重试机制

- 所有错误至少 **5 次重试**（指数退避）
- 指数退避：1s → 2s → 4s → 8s → 16s

### 错误记录

所有错误写入：
- `logs/pipeline-YYYY-MM-DD.log` — 运行日志
- `logs/failures.json` — 失败记录（原因、时间戳、关键词、重试次数）

### 提醒通知

- **终端通知**：macOS `osascript` 通知
- **日志文件**：每个错误详细记录

### 失败场景处理

| 场景 | 处理方式 |
|------|----------|
| Keyword Planner API 失败 | 重试 5 次 → 失败记录 → 使用预设数据继续 |
| Google Trends 失败 | 重试 5 次 → 失败记录 → 跳过该数据源 |
| Exa 搜索无结果 | 重试 5 次 → 失败记录 → 标记需人工审核 |
| 学术搜索失败 | 跳过学术引用 → 继续使用其他数据源 |
| 质量检查不通过 | 自动重写，最多 3 次 → 仍失败通知你 |
| Git push 冲突 | 自动 stash + pull + push，仍失败则通知你 |
| Vercel 部署失败 | 记录 → 通知你 → 提供 revert 方案 |

## 10. 文件结构

```
wag-frontend/
├── content/blog/                  # 正式文章 MDX
├── logs/
│   ├── pipeline-YYYY-MM-DD.log    # 运行日志
│   └── failures.json              # 失败记录
├── agents/
│   ├── master.ts                  # Master Agent
│   ├── researcher.ts              # Researcher Agent (可并行多个)
│   ├── drafter.ts                 # Drafter Agent
│   └── publisher.ts               # Publisher Agent
├── lib/
│   ├── keyword-planner.ts         # Google Keyword Planner API
│   ├── google-trends.ts           # Google Trends API
│   ├── exa.ts                     # Exa 搜索接口
│   ├── scholar.ts                 # Semantic Scholar API
│   ├── originality.ts             # 原创度检查（Exa + diff-match-patch）
│   ├── ai-detector.ts             # AI 率检测
│   ├── retry.ts                   # 重试逻辑（5次指数退避）
│   └── notify.ts                  # 通知机制

    ├── run.ts                     # 流水线入口
    └── keywords.ts               # 预设关键词库 + 竞品列表
```
