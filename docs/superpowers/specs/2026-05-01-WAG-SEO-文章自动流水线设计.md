**业务背景**：Winning Adventure Global (WAG) 帮助澳大利亚企业从中国采购，需要持续输出高质量 SEO 内容建立信任。

## 3. Agent 职责

| Agent | 职责 | 使用工具 |
|-------|------|----------|
| **MASTER** | GSC 扫描 → 任务分配 → 流程控制 | GSC API, Claude Code |
| **RESEARCHER 1-2** | 并行研究（关键词扩展 + Exa + 学术搜索） | Exa, Semantic Scholar API |
| **DRAFTER** | 合并研究 → 生成 MDX（低AI率优化） | Claude API, 本地写入 |
| **PUBLISHER** | 创建 MDX 文件 → git commit → git push | Git, Vercel |

## 4. 数据源

### 免费数据源

| 来源 | 用途 |
|------|------|
| **Google Search Console API** | 关键词曝光/点击数据，发现机会词 |
| **Exa API** | 实时网络搜索，行业/竞品研究 |
| **Semantic Scholar API** | 学术研究引用，提升 E-E-A-T |
| **预设核心关键词库** | 种子词，AI 扩展成长尾词队列 |

### 数据流

```
预设核心词 → MASTER 扩展长尾词队列
    ↓
GSC API 扫描 → 发现关键词机会
    ↓
MASTER 分配任务 → RESEARCHER Agent 并行研究
    ↓
研究摘要池 → DRAFTER Agent 生成 MDX
    ↓
PUBLISHER Agent → Git push → Vercel 部署
```

## 5. 低 AI 率策略

每篇文章必须：

1. **包含真实数据/案例** — 非通用描述，从 Exa 搜索获取
2. **混合句式长度** — 长短句交错，避免 AI 典型均匀分布
3. **融入 WAG 具体服务细节** — 真实的服务描述、价格范围、流程
4. **故意留有小瑕疵** — 非常微小的"人类痕迹"（可选）
5. **时效性内容** — Exa 实时搜索确保内容最新

## 6. 错误处理策略

### 重试机制

- 所有错误至少 **5 次重试**（指数退避）
- 指数退避：1s → 2s → 4s → 8s → 16s

### 错误记录

所有错误写入：
- `logs/pipeline-YYYY-MM-DD.log` — 运行日志
- `logs/failures.json` — 失败记录（包含原因、时间戳、关键词）

### 失败场景处理

| 场景 | 处理方式 |
|------|----------|
| GSC API 失败 | 重试 5 次 → 失败记录 → 跳过本次运行 |
| Exa 搜索无结果 | 重试 5 次 → 失败记录 → 标记需人工审核 |
| 学术搜索失败 | 跳过学术引用 → 继续使用其他数据源 |
| DRAFTER 生成失败 | 重试 5 次 → 失败记录 → 跳过该篇 |
| Git push 冲突 | 自动 stash + pull + push，仍失败则通知你 |
| Vercel 部署失败 | 记录 → 通知你 → 提供 revert 方案 |

### 提醒通知

- **终端通知**：macOS `osascript` 通知
- **日志文件**：每个错误详细记录
