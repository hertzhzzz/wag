## 1. 背景与目标

**业务背景**：Winning Adventure Global (WAG) 帮助澳大利亚企业从中国采购，需要持续输出高质量 SEO 内容建立信任。

## 3. Agent 职责

| Agent | 职责 | 使用工具 |
|-------|------|----------|
| **MASTER** | 关键词优先级排序 → 任务分配 → 流程控制 | Keyword Planner API, Exa, Claude Code |
| **RESEARCHER 1-2** | 并行研究（Exa 搜索 + 学术搜索） | Exa, Semantic Scholar API |
| **DRAFTER** | 合并研究 → 生成 MDX（低AI率优化） | Claude API, 本地写入 |
| **PUBLISHER** | 创建 MDX 文件 → git commit → git push | Git, Vercel |

## 4. 数据源

### 免费数据源

| 来源 | 用途 | 备注 |
|------|------|------|
| **Google Keyword Planner API** | 搜索量、趋势、相关关键词 | 需要 Google Ads 账号，免费 |
| **Exa API** | 实时网络搜索、行业研究、竞品分析 | 免费额度：MCP 150次/天 |
| **Semantic Scholar API** | 学术研究引用，提升 E-E-A-T | 免费，100M+ 论文 |
| **预设核心关键词库** | 种子词，Keyword Planner 扩展 | WAG 业务相关核心词 |

### 数据流

```
预设核心词库 → Google Keyword Planner → 获取搜索量 + 趋势
    ↓
Exa API → 发现话题趋势 + 竞品覆盖 + 新闻
    ↓
MASTER 综合判断优先级 → 关键词机会队列
    ↓
MASTER 分配任务 → RESEARCHER Agent 并行研究
    ↓
研究摘要池 → DRAFTER Agent 生成 MDX
    ↓
PUBLISHER Agent → Git push → Vercel 部署
```

## 5. 低 AI 率策略

每篇文章必须：

1. **包含真实数据/案例** — 非通用描述，从 Exa 搜索获取最新行业数据
2. **混合句式长度** — 长短句交错，避免 AI 典型均匀分布（低 burstiness）
3. **融入 WAG 具体服务细节** — 真实的服务描述、价格范围、流程
4. **使用 hedging 语言** — "I think," "in my experience," "honestly" 等
5. **打破公式化结构** — 不是每个 H2 都按"定义 → 好处 → 步骤 → 总结"
6. **时效性内容** — Exa 实时搜索确保内容最新（6个月内数据）

**目标**：AI 率 < 30%，通过工具检测（GPTZero、Turnitin 等）验证。

## 6. 错误处理策略

### 重试机制

- 所有错误至少 **5 次重试**（指数退避）
- 指数退避：1s → 2s → 4s → 8s → 16s

### 错误记录

所有错误写入：
- `logs/pipeline-YYYY-MM-DD.log` — 运行日志
- `logs/failures.json` — 失败记录（包含原因、时间戳、关键词、重试次数）

### 失败场景处理

| 场景 | 处理方式 |
|------|----------|
| Keyword Planner API 失败 | 重试 5 次 → 失败记录 → 使用预设数据继续 |
| Exa 搜索无结果 | 重试 5 次 → 失败记录 → 标记需人工审核 |
| 学术搜索失败 | 跳过学术引用 → 继续使用其他数据源 |
| DRAFTER 生成失败 | 重试 5 次 → 失败记录 → 跳过该篇 |
| Git push 冲突 | 自动 stash + pull + push，仍失败则通知你 |
| Vercel 部署失败 | 记录 → 通知你 → 提供 revert 方案 |

### 提醒通知

- **终端通知**：macOS `osascript` 通知
- **日志文件**：每个错误详细记录
